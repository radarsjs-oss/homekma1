# 레이더 추적기 정의와 역할
## 레이더 시스템의 핵심 구성 요소 및 기본 기능
- 관측된 레이더 신호를 동일 표적의 항적으로 연관시켜 정보를 추출하는 지휘 통제 시스템의 필수 장치임
- 여러 타겟 신호 중 유효한 데이터를 식별하고 다수 센서에서 수집된 정보를 하나로 통합하는 데이터 융합 수행함
- 연속적인 관측치 업데이트를 통해 개별 표적의 현재 이동 속도와 진행 방향을 정밀하게 산출하는 능력을 보유함
- 수신 과정에서 발생하는 가짜 탐지 신호인 false alarm을 효과적으로 걸러내어 추적 신뢰도를 보장하는 역할임
- 개별 레이더 측정치에 포함된 오차를 수학적으로 보정하여 매끄러운 항적 곡선을 생성하는 필터링 작업을 담당함

::: notes
레이더 추적기는 감시 레이더가 보고하는 '플롯'들을 분석하여 실제 표적의 항적을 생성하는 소프트웨어 또는 하드웨어 모듈입니다. 단순히 위치를 표시하는 것을 넘어, 속도와 방향 같은 동적인 정보를 계산하고 잡음을 제거하는 중추적인 역할을 수행합니다.
:::

# 항적 정보 구성
## 추적 데이터의 필수 및 부가 요소 체계
- 표적의 공간상 위치를 나타내는 2차원 또는 3차원 좌표 정보를 포함하여 실시간 경로를 정의함
- 목표물이 이동하는 방위인 헤딩 정보와 초당 이동 거리를 나타내는 속도 데이터를 핵심 수치로 제공함
- 시스템 내에서 각 표적을 고유하게 식별하기 위해 할당된 고유 트랙 번호를 부여하여 개별 관리함
- 피아식별 시스템이나 보조 감시 레이더를 통해 획득한 IFF 모드 정보 및 호출 부호 정보를 포함함
- 현재 산출된 항적 데이터의 신뢰도 수준이나 오차 범위를 나타내는 통계적 불확실성 수치를 함께 명시함

[표 1: 레이더 항적 구성 데이터 종류]
| 데이터 범주 | 주요 항목 | 필수 여부 |
| :--- | :--- | :--- |
| 기본 기구학 | 위치, 속도, 헤딩 | 필수 |
| 관리 식별 | 트랙 번호, 유효성 | 필수 |
| 부가 정보 | IFF 모드, 고도, 호출부호 | 선택 |
| 신뢰도 지표 | 공분산 행렬, 오차 범위 | 선택 |

::: notes
항적은 단순한 위치 점의 집합이 아니라, 식별 번호와 운동 정보, 그리고 신뢰도 수치를 포함한 종합적인 정보 패키지입니다. 이를 통해 관제사는 표적이 누구인지, 어디로 가는지, 그 정보가 얼마나 정확한지 판단할 수 있습니다.
:::

# 추적 처리 흐름
## 표준적인 레이더 추적 알고리즘 수행 단계
- 새롭게 입력된 레이더 플롯을 기존 항적 리스트와 대조하여 가장 적합한 대상을 찾는 연관 단계를 수행함
- 선택된 측정값을 바탕으로 이전 예측치를 수정하여 표적 상태를 최신화하는 항적 평활화 과정을 진행함
- 기존 항적에 할당되지 않은 잔여 플롯들을 대상으로 새로운 잠재적 항적을 생성하는 항적 개시를 수행함
- 업데이트가 없거나 탐지 영역을 벗어난 항적의 상태를 예측하거나 종료 여부를 결정하는 유지 관리를 함
- 한정된 연산 자원을 고려하여 수백 개의 표적을 실시간으로 동시 처리할 수 있는 효율적인 로직을 적용함

[ASCII 인포그래픽: 추적 루프]
[레이더 입력] -> [데이터 연관] -> [필터링/평활화] -> [상태 예측] -> [항적 관리] -> [결과 출력]

::: notes
모든 레이더 추적기는 연관, 갱신, 개시, 유지라는 공통된 네 단계를 반복합니다. 특히 이전 위치를 기반으로 다음 위치를 예측하고, 실제 들어온 값과 비교하여 오차를 줄여나가는 과정이 추적 기술의 핵심입니다.
:::

# 데이터 연관 기법
## 플롯과 항적 사이의 통계적 연결 원리
- 예측된 표적 위치 주변에 일정한 통계적 영역인 수용 게이트를 설정하여 후보 플롯군을 1차적으로 선별함
- 설정된 게이트 내에서 예측 지점과 물리적 거리가 가장 가까운 플롯을 선택하는 최근접 이웃 방식을 사용함
- 클러터가 심한 환경에서 여러 플롯의 확률적 가중치를 계산하여 결합하는 PDAF 또는 JPDAF 알고리즘을 적용함
- 연관된 플롯 정보를 사용하여 필터 내부의 예측 오차를 보정하고 다음 단계의 상태 추정치를 갱신함
- 잘못된 연관으로 인해 항적이 튀는 현상을 방지하기 위해 통계적 유의미성을 지속적으로 검증하는 로직을 포함함

[예시: 밀집 클러터 환경의 데이터 연관]
강한 해면 클러터 상황에서 단순 최근접 방식은 가짜 신호를 선택할 확률이 높으므로, 주변의 모든 유효 플롯을 확률적으로 고려하는 JPDAF를 적용하여 추적 안정성을 확보함

::: notes
데이터 연관은 수많은 레이더 신호 중 어느 것이 실제 내 타겟인지 결정하는 과정입니다. 복잡한 전장 상황이나 노이즈가 많은 곳에서는 단순히 가까운 것을 잡는 것보다 확률적인 모델을 사용하는 것이 훨씬 유리합니다.
:::

# 항적 관리 체계
## 신규 항적 개시 및 종료 판단 기준
- 미연관 플롯에 대해 즉시 확정 항적을 생성하지 않고 일시적으로 관찰하는 잠정 항적 상태를 먼저 부여함
- 최근 N번의 스캔 중 M번 이상 탐지되어야 실제 표적으로 인정하는 M-of-N 규칙을 통해 가짜 항적을 억제함
- 탐지 신뢰도가 높아져 오차 공분산 행렬의 크기가 임계치 이하로 감소할 때 항적을 운영자에게 표시함
- 연속적인 미탐지 횟수가 기준을 초과하거나 위치 불확실성이 너무 커지면 해당 항적을 시스템에서 삭제함
- 표적이 일시적으로 가려지는 경우에도 이전 정보를 바탕으로 위치를 예측하여 항적 번호를 유지하는 기능을 함

[표 2: 항적 상태 전이 기준 예시]
| 상태 변환 | 조건 (Rule) | 목적 |
| :--- | :--- | :--- |
| 신규 -> 잠정 | 미연관 플롯 발생 | 후보군 관리 |
| 잠정 -> 확정 | 3-of-5 탐지 성공 | 허위 항적 제거 |
| 확정 -> 예측 | 1회 미탐지 발생 | 추적 연속성 유지 |
| 확정 -> 삭제 | 3회 연속 미탐지 | 자원 최적화 |

::: notes
항적 관리는 레이더 화면이 깨끗하게 유지되도록 돕는 관리자 역할을 합니다. 가짜 신호가 들어오면 일단 지켜보다가 확실해지면 번호를 붙여주고, 표적이 사라지면 신속하게 지워서 시스템 부하를 줄입니다.
:::

# 칼만 필터 원리
## 선형 시스템에서의 최적 상태 추정 알고리즘
- 현재 상태의 통계적 예측치와 새로운 측정치의 가중 평균을 통해 오차를 최소화하는 재귀적 필터 구조임
- 시스템의 물리적 운동 모델을 정의하는 상태 방정식과 측정값의 관계를 나타내는 관측 방정식을 기본으로 함
- 예측값의 불확실성과 측정 잡음의 크기를 비교하여 신뢰도를 조절하는 칼만 이득 계수를 실시간으로 계산함
- 모든 잡음이 평균이 0인 가우시안 분포를 따른다는 가정하에 최소 제곱 오차를 보장하는 수학적 해법임
- 표적의 기동으로 인해 필터가 예측에 과신하는 현상을 막기 위해 인위적으로 공분산을 조절하는 기법을 포함함

[ASCII 인포그래픽: 칼만 필터 루프]
[이전 추정치] -> [시간 갱신(예측)] -> [칼만 이득 계산] -> [측정 갱신(보정)] -> [최종 추정치]

::: notes
칼만 필터는 현대 추적 기술의 심장입니다. 필터는 끊임없이 "내가 아는 값"과 "새로 들어온 값" 사이에서 줄타기를 하며 가장 정확한 값을 찾아냅니다. 특히 과거의 모든 데이터를 저장할 필요 없이 직전 값만 있으면 된다는 점이 강력합니다.
:::

# 확장 칼만 필터
## 비선형 레이더 시스템 대응을 위한 확장 기술
- 현실의 비선형적 물리 현상을 처리하기 위해 기존 칼만 필터를 선형화 과정으로 개선한 알고리즘임
- 비선형 함수를 특정 지점에서 1차 테일러 급수로 전개하여 국부적으로 선형적인 모델 행렬을 도출함
- 매 단계의 추정값을 기준으로 편미분을 수행하여 야코비안 행렬을 동적으로 갱신하며 상태를 추종함
- 직교 좌표계와 구 좌표계 사이의 변환과 같이 복잡한 수식이 포함된 레이더 관측 모델을 효율적으로 처리함
- 초깃값 오차가 클 경우 선형화 기준점이 실제와 달라져 알고리즘이 발산할 수 있으므로 주의가 필요함

[표 3: 선형 KF와 확장 KF 비교]
| 구분 | 선형 칼만 필터 (KF) | 확장 칼만 필터 (EKF) |
| :--- | :--- | :--- |
| 시스템 모델 | 선형 행렬 ($A, H$) | 비선형 함수 ($f, h$) |
| 수학적 기법 | 단순 행렬 연산 | 테일러 급수/야코비안 |
| 적용 대상 | 직선 운동, 단순 센서 | 회전 기동, 구좌표 센서 |
| 연산 복잡도 | 낮음 | 중간 (미분 필요) |

::: notes
대부분의 레이더는 거리와 각도를 측정하지만, 우리가 필요한 것은 지도상의 X, Y 좌표입니다. 이 변환 과정은 비선형적이라 일반 칼만 필터로는 한계가 있습니다. EKF는 이를 미분을 통해 살짝 직선처럼 펴서 계산하는 영리한 방식을 씁니다.
:::

# EKF 레이더 모델
## 상태 변수 정의 및 수식적 구현 사례
- 표적의 수평 거리, 이동 속도, 고도를 상태 변수 벡터로 설정하여 3차원 공간상의 거동을 추적함
- 시간에 따른 위치 변화율을 속도로 정의하고 고도와 속도가 일정하다는 물리적 등속 운동 모델을 적용함
- 피타고라스 정리에 기반한 거리 산출 공식을 관측 함수로 정의하여 실제 레이더 측정값과 대조함
- 측정 잡음과 시스템 프로세스 잡음의 공분산 행렬을 설정하여 필터의 반응 감도를 조절하는 설계를 함
- MATLAB 등 소프트웨어를 활용하여 예측, 이득 계산, 추정치 갱신 과정을 재귀적 함수 형태로 구현함

[예시: 레이더 측정 모델 수식화]
상태 벡터 $x = [pos, vel, alt]^T$ 일 때, 관측치 $z$는 $\sqrt{pos^2 + alt^2} + v$ 로 모델링되며, 이를 미분하여 야코비안 행렬 $H$를 산출함

::: notes
실제 시스템을 만들 때는 대상을 위치, 속도, 고도 같은 벡터로 정의해야 합니다. 레이더가 쏜 전파가 돌아온 시간을 통해 거리를 구하고, 이를 다시 수학 모델에 넣어 우리가 측정하지 않은 속도까지 알아내는 과정이 담겨 있습니다.
:::

# 적응형 확장 칼만 필터
## 거리 변화에 따른 잡음 모델의 동적 조정
- 고정된 잡음 수치를 사용하는 대신 표적과의 거리에 따라 실시간으로 잡음 공분산을 수정하는 기술임
- 수신 신호 증폭 과정에서 발생하는 열잡음이 거리의 제곱에 비례하여 증가하는 물리적 특성을 반영함
- 표적 산란체의 간섭으로 인한 글린트 잡음이 거리가 멀어질수록 감소하는 반비례 모델을 결합함
- 열잡음 모델과 글린트 잡음 모델의 합을 통해 현재 거리에 최적화된 통합 잡음 공분산을 산출함
- 거리 변화가 극심한 장거리 비행체 추적 시 필터의 발산을 막고 일관된 추적 정밀도를 유지하게 함

[ASCII 인포그래픽: 적응형 잡음 모델링]
[거리(R) 입력] -> [열잡음($R^2$) 계산] + [글린트($1/R^2$) 계산] -> [최적 공분산 $R$ 갱신] -> [필터 적용]

::: notes
레이더는 가까이 있을 때와 멀리 있을 때 들어오는 잡음의 성질이 완전히 다릅니다. 고정된 필터는 한쪽 상황에서만 잘 맞지만, 적응형 필터(AEKF)는 거리를 보고 스스로 필터의 강도를 조절하여 언제나 최상의 성능을 냅니다.
:::

# AEKF 실무 사례
## KSLV-I 한국형 발사체 추적 성능 분석
- 한국형 발사체의 실제 비행 궤적 데이터를 기반으로 AEKF 알고리즘의 유효성을 컴퓨터로 검증함
- 이륙부터 궤도 진입까지 급격한 고도 및 거리 변화가 발생하는 전 과정을 시뮬레이션 시나리오로 구성함
- 기존 EKF 대비 방위각과 고도각의 오차 평균값을 0에 가깝게 유지하여 추적 정밀도를 개선함
- 거리가 멀어질수록 고정형 EKF에서 발생하는 오차 누적 현상인 drift를 효과적으로 억제함을 확인함
- AN/FPS-16 정밀 추적 레이더의 요구 사양인 200urad 수준의 정확도를 전 비행 구간에서 만족함

[표 4: KSLV-I 주요 비행 이벤트별 위치]

| :--- | :--- | :--- | :--- |
| 이륙 (Launch) | 0 | 0.1 | 0 |
| 음속 돌파 | 54 | 7.2 | 0.8 |

::: notes
나로호와 같은 우주 발사체는 엄청난 속도로 수천 킬로미터를 날아갑니다. 이때 거리에 따라 필터 설정을 바꾸지 않으면 추적을 놓치기 쉬운데, AEKF를 적용하면 전 구간에서 아주 정밀하게 위치를 쫓아갈 수 있다는 것이 증명되었습니다.

# 다중 모델 추정
## 다양한 기동 상태에 대응하는 IMM 알고리즘
- 단일 운동 모델의 한계를 극복하기 위해 서로 다른 특성을 가진 여러 필터를 병렬로 운용하는 방식임
- 표적이 등속 비행에서 가속 또는 선회 기동으로 전환하는 시점을 통계적으로 감지하여 적절히 대응함
- 각 모델의 예측값과 실제 측정값의 일치 정도를 우도로 계산하여 모델별 가중치를 실시간 갱신함
- 마르코프 연쇄 모델을 사용하여 기동 모드 간의 전환 확률을 정의하고 모델 간의 상호작용을 수행함
- 단순한 알고리즘 대비 연산량은 다소 증가하나 기동 표적에 대해 획기적인 추적 성능 향상을 제공함

[ASCII 인포그래픽: IMM 모델 구성]
      /-- [모델 1: 등속(CV)] --\
[입력] --- [모델 2: 가속(CA)] --- [가중치 결합] -> [최종 출력]
      \-- [모델 3: 선회(TURN)] --/

::: notes
표적은 늘 일직선으로만 가지 않습니다. 갑자기 꺾기도 하고 멈추기도 하죠. IMM은 내부에 여러 전문가(모델)를 두고, 지금 상황에 가장 잘 맞는 전문가의 목소리를 더 크게 반영하여 복잡한 움직임을 따라가는 기술입니다.
:::

# IMM 처리 단계
## 상호작용 및 확률 업데이트의 4단계 루프
- 이전 단계의 모델 확률을 기반으로 각 필터의 입력 상태와 공분산을 섞어주는 상호 혼합 단계를 거침
- 혼합된 정보를 바탕으로 각 모델 필터가 독립적으로 예측 및 측정 업데이트를 수행하여 상태를 산출함
- 측정 잔차의 크기를 분석하여 현재 표적이 어떤 모델에 가장 가까운지 나타내는 모델 확률을 갱신함
- 업데이트된 모델 확률을 가중치로 사용하여 모든 필터의 결과물을 하나로 합치는 상태 결합을 함
- 기동이 감지되면 기동 모델의 확률이 급격히 상승하여 필터가 민첩하게 반응하도록 구조적으로 유도함

[표 5: IMM 알고리즘 주요 수식 개념]
| 단계 | 주요 작업 | 통계적 의미 |
| :--- | :--- | :--- |
| 1. Mixing | 상태/공분산 혼합 | 모드 전환 가능성 선반영 |
| 2. Filtering | 개별 KF 갱신 | 각 가설별 최적 추정 |
| 3. Prob. Update | 우도(Likelihood) 계산 | 현재 모델 적합도 평가 |
| 4. Combining | 가중 합산 | 최종적인 통합 상태 결정 |

::: notes
IMM은 단순히 여러 필터를 돌리는 게 아니라, 그 결과를 섞고 나누는 과정을 반복합니다. 타겟이 선회하면 선회 모델 필터가 "이건 내 차례다!"라고 주장하며 확률이 올라가고, 전체 시스템이 그 모델의 예측을 더 믿게 되는 원리입니다.
:::

# IMM 운동 모델
## 추적 정밀도 향상을 위한 핵심 모델 정의
- 가속도가 없는 직선 비행 구간에서 잡음을 최소화하고 안정적인 정보를 제공하는 등속도 모델임
- 속도가 일정하게 변화하는 기동 상황을 수용하기 위해 가속도 성분을 상태 변수에 포함한 등가속도 모델임
- 수평 또는 3차원 공간에서 일정한 속력으로 방향을 바꾸는 기동을 정밀 추종하는 조항 선회 모델임
- 선회 시 가속도 벡터가 속도 벡터에 수직이어야 한다는 기구학적 제약을 활용하여 오차를 줄임
- 각 모델에 부여된 프로세스 잡음 $Q$ 값을 차별화하여 정밀도와 반응 속도 사이의 균형을 최적화함

[예시: 복합 기동 시나리오 대응]
항공기가 직선 비행(CV 모델 우세)을 하다가 갑자기 급선회(TURN 모델 확률 급상승)에 돌입할 때, IMM은 모델 간 가중치를 즉각 조절하여 항적 유실을 방지함

::: notes
IMM의 실력은 내부에 어떤 모델을 두느냐에 달렸습니다. 보통 직선 비행용, 가속용, 회전용 모델을 세트로 넣습니다. 특히 회전 모델은 항공기가 속력을 유지하며 꺾는다는 물리적 제약 조건을 수식에 넣어 정확도를 높입니다.
:::

# IMM 성능 분석
## 단일 필터 대비 기동 추종 성능 비교 결과
- 직선 비행 후 5.6g 급선회를 수행하는 가혹한 시나리오에서 단일 모델 대비 획기적 오차 감소를 보임
- 기동이 시작되는 순간 발생하는 시간 지연에 의한 래그 오차를 다중 모델 구조를 통해 빠르게 회복함
- 기동 종료 후 다시 안정적인 비기동 구간으로 진입할 때 오차 수렴 속도가 단일 모델보다 월등히 빠름
- 전환 확률 행렬이나 잡음 파라미터의 미세한 설정 변화보다 적절한 모델 선정 자체가 성능을 결정함
- 3차원 다이빙 기동 시나리오에서도 TURN 모델을 포함한 IMM이 가장 낮은 RMS 위치 오차를 나타냄

[표 6: 피크 RMS 오차 비교 (Scan 레이더)]
| 시나리오 | CV 단일 | CA 단일 | CV-CA-TURN IMM |
| :--- | :--- | :--- | :--- |
| 비기동 (Steady) | 289 / 42 | 360 / 133 | **241 / 22** |
| 5.6g 선회 | 968 / 323 | 445 / 179 | **391 / 120** |
| 다이빙 기동 | 246 / 206 | 180 / 136 | **167 / 129** |

::: notes
성능 평가 결과, IMM은 모든 면에서 단일 필터를 압도합니다. 특히 기동 중에도 오차가 적지만, 기동이 끝난 뒤 다시 제자리(안정 상태)를 찾는 능력이 탁월합니다. 이는 실제 전장에서 표적을 놓치지 않고 끝까지 잡고 있는 힘이 됩니다.
:::

# 4D 레이더 융합
## 악천후 대응을 위한 라이다 센서 결합 기술
- 전파를 사용하여 눈, 비 등 기상 조건에 관계없이 표적 탐지가 가능한 4D 레이더의 장점을 활용함
- 고밀도 기하학 정보를 제공하나 악천후에 취약한 라이다 센서의 단점을 레이더로 보완하는 융합 구조임
- 각 센서의 측정치를 개별 IMM 필터에 먼저 적용한 후 상태값을 통합하는 Late fusion 기법을 제안함
- 복잡한 도심 주행 환경에서 차량이나 보행자의 기동을 정확히 예측하기 위해 IMM 엔진을 핵심으로 사용함
- 실제 악천후 데이터가 포함된 K-Radar 데이터셋 환경에서 단일 센서 대비 압도적인 추적 강건성을 입증함

[ASCII 인포그래픽: Late Fusion 구조]
[4D 레이더] -> [IMM 필터 A] --\
                               [상태/공분산 결합] -> [최종 항적]
[라이다 센서] -> [IMM 필터 B] --/

::: notes
자율주행차에서는 날씨가 큰 변수입니다. 라이다는 맑을 때 좋고, 레이더는 비 올 때 강하죠. 이 두 정보를 Late Fusion 방식으로 합치고 IMM 필터로 다듬으면, 어떤 날씨에서도 차량이나 사람의 움직임을 놓치지 않는 튼튼한 눈이 됩니다.
:::

# 딥러닝 기반 추적
## LSTM 네트워크를 활용한 모델 프리 접근법
- 복잡한 물리 수식 기반의 운동 모델 정의 없이 데이터 학습을 통해 타겟 거동을 직접 추정하는 방식임
- 긴 의존성을 처리하는 LSTM의 시계열 학습 능력을 활용하여 표적의 과거 경로로부터 미래를 예측함
- 수학적 모델의 불완전성으로 인해 발생하는 기존 칼만 필터의 성능 저하 문제를 데이터로 해결하고자 함
- 데이터 연관 모듈에는 양방향 LSTM을, 필터링과 예측에는 적층형 LSTM 모델을 각각 배치하여 최적화함
- 센서 잡음이 심한 환경에서도 수천 개의 학습 경로 데이터를 통해 강건한 상태 추정 성능을 확보함

[표 7: MTT 딥러닝 모듈별 기능]
| 모듈 명칭 | 역할 설명 | 적용 모델 |
| :--- | :--- | :--- |
| Association | 측정치-항적 일치 판단 | Bi-Directional LSTM |
| Filter | 측정치 잡음 제거/정제 | 3-Layer Stacked LSTM |
| Prediction | 차기 시점 상태 예측 | 3-Layer Stacked LSTM |
| Management | 항적 생성 및 소멸 관리 | Rate-based Logic |

::: notes
최근에는 수학 공식 대신 인공지능을 씁니다. LSTM은 과거의 움직임을 기억했다가 다음에 어디로 갈지 맞히는 데 탁월합니다. 공식이 복잡한 군사용 기동이나 변칙적인 움직임도 수많은 데이터를 미리 보여주면 인공지능이 스스로 요령을 터득합니다.
:::

# 딥러닝 추적 아키텍처
## 모듈형 다중 표적 추적 시스템 설계
- 전체 추적 시스템을 연관, 필터링, 예측 등 독립된 기능 블록으로 구성하여 시스템 해석력을 높임
- 수많은 측정 데이터가 들어올 경우 연관 모듈의 부하를 줄이기 위해 게이팅 블록에서 클러스터링을 수행함
- 레이더와 같은 비협조 센서뿐만 아니라 IFF, AIS 등 협조 센서 데이터도 수용 가능한 유연한 구조임
- 각 기능 모듈을 개별적으로 훈련시키거나 특정 상황에 맞춰 교체할 수 있는 높은 확장성을 제공함
- 50만 개 이상의 랜덤 트랙 데이터를 통해 학습된 모델은 복합 기동 상황에서도 높은 정확도를 유지함

[예시: 복합 기동 시나리오 학습 결과]
직선, 선회, 가속, 나선형 비행이 섞인 트랙에 대해 LSTM 모델이 IMM 필터 대비 위치 예측 오차를 약 71.6% 감소시킴

::: notes
전체 시스템을 하나로 뭉치지 않고 '조립식'으로 만드는 것이 요즘 추세입니다. 측정을 정제하는 모듈, 미래를 보는 모듈 등을 따로 만들어 성능을 높입니다. 이 방식은 나중에 새로운 센서가 추가되어도 그 부분만 갈아 끼우면 되니 매우 편리합니다.
:::

# 하이브리드 추적 필터
## 모델 기반과 데이터 기반의 결합 KalmanNet
- 전통적 칼만 필터의 구조적 안정성과 딥러닝의 학습 능력을 결합한 최신 하이브리드 아키텍처임
- 복잡한 공분산 행렬 계산 과정을 생략하고 GRU 네트워크가 데이터로부터 최적의 칼만 이득을 직접 도출함
- 시스템의 잡음 통계 정보를 사전에 알지 못해도 데이터 시퀀스를 통해 환경 특성을 스스로 파악함
- RNN의 내부 메모리를 활용하여 공분산의 통계적 정보를 간접적으로 추적하고 관리하는 메커니즘을 가짐
- 실제 도로 주행 데이터인 RadarScenes를 이용한 평가에서 데이터 기반 방식의 가능성과 한계를 검증함

[표 8: KalmanNet vs IMM 성능 지표 (8-drive)]
| 평가 지표 | KalmanNet (Hybrid) | IMM (Model-based) |
| :--- | :--- | :--- |
| 위치 RMSE (m) | 1.23 | **1.08** |
| 속도 RMSE (m/s) | 2.98 | **1.28** |
| 가속도 RMSE ($m/s^2$) | 17.96 | **13.84** |
| 데이터 강건성 | 중간 (안정성 과제) | **높음 (검증됨)** |

::: notes
KalmanNet은 뼈대는 수학(칼만 필터)을 쓰되, 알맹이인 '칼만 이득'만 인공지능이 계산하게 만든 하이브리드입니다. 하지만 실제 차량 데이터로 테스트해 보니 아직은 정통파인 IMM 필터가 더 정확하고 안정적이었습니다. 인공지능이 갈 길이 아직 좀 남았다는 뜻이죠.
:::

# 추적 기술 총괄 비교
## 알고리즘별 특성 및 최적 적용 분야 요약
- 단순 선형 운동 및 저사양 시스템에는 연산 부하가 가장 적은 고정 이득 또는 선형 칼만 필터가 적합함
- 구좌표 기반의 단일 표적 정밀 추적에는 비선형 대응력이 우수한 EKF 또는 적응형 EKF가 널리 쓰임
- 고속 기동기가 포함된 다중 표적 환경에서는 IMM 알고리즘이 성능과 효율성 측면에서 가장 검증된 해법임
- 모델 정의가 어렵고 데이터가 풍부한 복합 시나리오에는 LSTM 등 딥러닝 기반 모듈형 추적기가 유망함
- 안전이 최우선인 자율주행 ADAS 시스템에는 아직 모델 기반의 IMM이나 정밀 튜닝된 필터가 신뢰성 면에서 우위임

[표 9: 추적 알고리즘 종합 비교표]
| 알고리즘 | 복잡도 | 기동 대응 | 주요 장점 | 한계점 |
| :--- | :--- | :--- | :--- | :--- |
| EKF | 중간 | 보통 | 보편적, 실시간성 | 비선형 오차 누적 |
| IMM | 높음 | 우수 | 다중 기동 최적화 | 파라미터 튜닝 필요 |
| LSTM | 매우 높음 | 매우 우수 | 모델 프리, 정밀함 | 학습 데이터 의존 |
| AEKF | 중간 | 보통 | 거리별 잡음 대응 | 특정 환경 특화 |

::: notes
이제 결론입니다. 상황에 맞는 연장을 골라야 합니다. 일반적인 상황엔 EKF, 복잡하게 움직이는 적군을 잡을 땐 IMM, 데이터가 넘쳐나고 미래 기술을 시험할 땐 LSTM을 씁니다. 현재 실무에선 검증된 IMM이 가장 사랑받고 있습니다.
:::
